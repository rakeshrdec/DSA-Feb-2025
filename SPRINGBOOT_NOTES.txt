now if we use(paymentService) this in our main method to reduce tight coupling so then IoC gets confused that which beans i have to use e and throws error like expecting 1 beans got 2

A--> To overcome this problem we can put the value which bean want to use in *application.properties file and then put * @ConditionalOnProperty(name = "{var_used_in_app.properties_file}", havingValue = "{value_to_run_this_bean}") and It only caused Ambiguity problem

this is how we can tell IoC to which bean t use or otherwise we have to recompile the class by removing @COntroller basically we want say here that don't use this (any ne either RazorPay or StripePay) as Bean then IoC/Spring  will not be confused
